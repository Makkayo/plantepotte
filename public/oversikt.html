📖 Først: noen begreper du må kunne

Hvis dette er nytt, les disse kortene før du fortsetter. Hver av dem dukker opp i diagrammene under.

🔋 Spenning vs. signal

Spenning (strøm) driver komponenter — som å gi en motor bensin. Signal forteller dem hva de skal gjøre — som en sjåfør. ESP32 sender små signaler som styrer mye større strømmer.

Eks: ESP32 sender et signal på 3.3V til MOSFET, som slipper gjennom 12V til LED.

⏚ GND (ground / minus)

Alt elektrisk må ha en vei tilbake til strømkilden. Det er GND. ALLE komponenter må dele samme GND — én lang skinne på breadboardet. Uten dette fungerer ingenting.

Hver eneste komponent får en GND-ledning til samme rad.

📍 GPIO (pinner på ESP32)

Hver pinne har et nummer (GPIO 26, GPIO 4, osv). Du forteller koden hvilken pinne du har koblet en sensor til, så vet ESP32 hvor den skal lese eller skrive.

GPIO 26 styrer MOSFET. GPIO 4 leser DHT22. GPIO 21/22 er for I2C.

⚡ PWM (Pulse-Width Modulation)

ESP32 kan ikke gi "halv strøm", men den kan blinke veldig fort — 1000 ganger i sekundet. Hvis lyset er på 30% av tiden, ser det dimmet ut for øyet. Det er PWM.

Slider på 70% i app-en → ESP32 sender PWM 70% → LED lyser 70%.

🔻 Buck converter

Tar høyere spenning inn og gir lavere ut. Vår tar 12V og lager 5V. Du stiller den med et potmeter og en multimeter til den viser nøyaktig 5.0V.

Uten dette: 12V på ESP32 = ødelagt brett.

🔌 MOSFET

Elektronisk bryter. Et lite signal på "SIG"-pinnen får MOSFET til å slippe gjennom stor strøm mellom VIN og OUT. Vår type (LR7843) trenger bare 3.3V signal — perfekt for ESP32.

ESP32 GPIO 26 → MOSFET SIG → LED slås av/på/dimmes.

📡 I2C (to-tråds buss)

En måte å snakke med flere chip-er på over kun to ledninger: SDA (data) og SCL (klokke). OLED-skjermen vår bruker dette.

GPIO 21 = SDA, GPIO 22 = SCL (standard på ESP32).

📊 ADC (analog-til-digital)

Noen sensorer (som jordfukt) gir en analog spenning som forteller hvor våt jorda er. ESP32 leser denne som et tall mellom 0 og 4095. Lavt tall = vått, høyt tall = tørt.

GPIO 34, 35, 32 er ADC-pinner og brukes til de tre jordfuktsensorene.

🧰 Breadboard

Prøveplate med hull. Hullene henger sammen i rader (5 horisontalt + lange skinner langs sidene). Sett komponenter ned, kobl med jumper-kabler — ingen lodding.

Bruk én skinne som 12V, én som 5V, én som 3.3V, én som GND.

Hvert fargede område er en komponent. Hvert farget segment er en ledning. Klikk en kabeltype i fargenøkkelen for å fremheve kun den typen, eller hold pekeren over en boks for mer info om komponenten.

📍 ESP32 — slik bruker du pinnene

ESP32-brettet har 30 pinner. Du bruker bare 14 (12 GPIO + 3.3V + GND). Grå = ledig. Rød = strapping-pinner som styrer oppstart — ikke koble noe til dem.

⚡ Strømtreet — fra vegg til chip

All strøm starter på 230V veggadapteren. Den splittes i to: 12V til LED-stripen, og 5V (via buck converter) til ESP32 og vannsensorene. ESP32 produserer selv 3.3V til alle de små sensorene.

🔌 230V AC vegguttak → ⚡ Adapter 12V 3A → 🔘 DC inline switch → ⚡ Slow-blow fuse 3A → 12V DC fordeling
├── 12V → MOSFET VIN → PWM-styrt → 💡 LED-strip
└── 12V → Buck converter → 5V justerbar
├── 5V → ESP32 VIN → ESP32 LDO intern → 3.3V
├── 3.3V → OLED
├── 3.3V → VL53L0X laser (deler I2C-buss med OLED)
├── 3.3V → DHT22
├── 3.3V → Jordfukt ×3
└── 3.3V → KY-040 (⚠ aldri 5V!)
└── 5V → ESP32-CAM (eget kort, kun strøm — data via WiFi)
⏚ GND-skinne: Alt som har strøm må også ha GND tilbake til samme punkt. Adapter, buck, MOSFET, ESP32 og hver eneste sensor — alle GND-kabler inn på én felles rad på breadboardet.

Klargjør 12V-strømkretsen — IKKE koble til ESP32 ennå

Bygg hele strømkjeden trinn for trinn. ESP32 skal ikke kobles til i dette steget.

⚡ Slik kobles dette fysisk (i rekkefølge):
  1. Adapter → DC inline switch: Plugg adapterens DC-plugg rett inn i hun-enden av vippebryteren. Ingen ledning.
  2. DC inline switch → barrel jack-pigtail: Plugg hann-enden av switchen inn i hun-socketen på den ene barrel jack-pigtailen. Du har nå skruklemmer tilgjengelig.
  3. Barrel jack-pigtail + → slow-blow fuse: Strip 10 mm bar ledning. Koble +-skruklemmen via Wago til det ene tråd-beinet på den leaded slow-blow-sikringen (T3A), og det andre beinet via en ny Wago til breadboard 12V-skinnen. Ingen holder — sikringen kobles direkte inline.
  4. Pigtail − terminal → GND-skinne: Koble −-skruklemmen direkte til breadboard GND-skinne.
  5. 12V-skinne → Buck converter: Koble barrel jack-pigtail (hann-plugg) inn i buck converterens inngang. Pigtailens skruklemmer kobles til 12V-skinne (+) og GND-skinne (−).
  6. Mål: Slå på bryteren. Bruk multimeter på buck converterens OUT+ og OUT−. Drei det blå potmeteret til du ser nøyaktig 5.0V. Har din modul ikke noe potmeter (fast 5V-modul)? Da bekrefter du bare at den viser ~5,0V (±0,2) — er den langt unna, er modulen feil og skal ikke brukes. Slå av bryteren.

🔧 Aldri brukt multimeter? (under ett minutt)
  1. Skru hjulet til likespenning (DC) — symbolet er V⎓ (V med rette streker over), ikke V∼ (det er vekselstrøm). Har måleren tallområder, velg 20.

OLED-skjerm (I2C)

📌 Gjør dette ÉN gang før trinn 4–7: mat 3.3V-skinnen ESP32 har bare én 3V3-pinne, men 6 enheter skal ha 3.3V (OLED, laser, DHT22, 3× jordfukt, KY-040). ⚠️ Viktig å huske
⚡

Still buck converter til 5V FØR tilkobling

Mål spenningen med multimeter og drei potmeteret til du ser 5.0V. For høy spenning ødelegger ESP32.

🎛️

KY-040 dreieknapp — bruk 3.3V, ikke 5V

Tåler bare 3.3V på VCC. 5V sender 5V tilbake på signal-pinnene og ødelegger ESP32-GPIO.

🔌

MOSFET SIG — aldri la den henge løs

Koble alltid SIG-pinnen til en GPIO. Flytende pinne gjør MOSFET halvåpen og overoppheter den.

🔗

Felles GND er ikke valgfritt

Alle komponenter MÅ dele samme minus/GND. Uten dette fungerer ikke signaler mellom ESP32 og resten.

💻

Koble fra VIN under programmering

Når ESP32 er koblet til PC via USB-C: ta av VIN-kabelen. To strømkilder samtidig kan skape problemer.

🕐

ESP32-tid er UTC — bruk TZ_OFFSET

ESP32 henter UTC-tid fra internett. Norge = +1 (vinter) eller +2 (sommer). Konstanten TZ_OFFSET_HOURS i config.py må endres to ganger i året.

📶

ESP32 støtter kun 2.4GHz WiFi

5GHz-nett vil ikke bli funnet. Sjekk router-innstillinger om du er usikker.

📊

Send rå ADC-verdier til Supabase

ESP32 skal sende rå tall (0–4095) for jordfukt, ikke prosent. Web-appen kalibrerer per sensor. Hvis ESP32 konverterer først, blir kalibrering umulig uten å flashe på nytt.

🌡️

DHT22 pull-up — din 3-pinners modul har den innebygd

DHT22 trenger en 4,7 kΩ pull-up mellom VCC og DATA. Men modulen din (3 pinner, kom med jumper-kabler) har den påloddet fra fabrikk → du trenger ikke gjøre noe. Bare en naken 4-pinners DHT22 krever at du selv setter 4,7 kΩ. Får du None/feil i Thonny, prøv å legge til motstanden.

🔑

Aldri del service_role-nøkkelen

Anon-nøkkelen (eyJ...) er trygg å ha i koden. Service_role-nøkkelen gir full DB-tilgang — aldri i kode, aldri del.

🚫

Ikke rør MørkeZ i Supabase

Supabase-prosjektet "MørkeZ" (udabzkunvagqavllcffw) er et eget prosjekt. Plantepotte og Matplanlegger bruker "Matplanlegger/Plantepotte" (ebjbxfwtwrahuokydvtj).

🐍 Koden (main.py)

# Pultlys — dimbar WS2812B-stripe styrt av KY-040 dreieknapp
# Helt lokalt: ingen WiFi, ingen app. Vri = dimme, trykk = av/på.
import time
from machine import Pin
import neopixel        # innebygd i MicroPython — ingen driver å laste opp

# ── Innstillinger (juster fritt) ──
ANTALL_LED = 30                # antall LED på stripa
DATA_PIN   = 13                # GPIO → stripas DIN (data-inn)
CLK_PIN    = 16                # KY-040 CLK
DT_PIN     = 17                # KY-040 DT
SW_PIN     = 18                # KY-040 SW (trykk-knapp)
GRUNNFARGE = (255, 170, 100)   # varmhvitt (R, G, B) ved full styrke
STEG       = 8                 # endring i lysstyrke per knapp-hakk
START_LYS  = 120               # lysstyrke ved oppstart (0–255)

# ── Hardware ──
np  = neopixel.NeoPixel(Pin(DATA_PIN), ANTALL_LED)
clk = Pin(CLK_PIN, Pin.IN, Pin.PULL_UP)
dt  = Pin(DT_PIN,  Pin.IN, Pin.PULL_UP)
sw  = Pin(SW_PIN,  Pin.IN, Pin.PULL_UP)

lys      = START_LYS           # nåværende lysstyrke 0–255
pa       = True                # er lyset på?
sist_lys = START_LYS           # husk styrken når vi slår av

def tegn():
    if pa and lys > 0:
        r = GRUNNFARGE[0] * lys // 255
        g = GRUNNFARGE[1] * lys // 255
        b = GRUNNFARGE[2] * lys // 255
    else:
        r = g = b = 0
    for i in range(ANTALL_LED):
        np[i] = (r, g, b)
    np.write()

clk_sist = clk.value()

def les_dreining():
    global lys, pa, clk_sist
    n = clk.value()
    if n != clk_sist and n == 0:          # CLK falt 1→0 = ett hakk
        if dt.value() != n:
            lys = min(255, lys + STEG)     # med klokka = lysere
        else:
            lys = max(0, lys - STEG)       # mot klokka = mørkere
        pa = True                          # det å vri vekker alltid lyset
        tegn()
    clk_sist = n

sw_sist = 1

def les_trykk():
    global pa, lys, sist_lys, sw_sist
    n = sw.value()
    if n == 0 and sw_sist == 1:           # nettopp trykket inn
        if pa:
            sist_lys = lys if lys > 0 else START_LYS
            pa = False
        else:
            pa = True
            if lys == 0:
                lys = sist_lys
        tegn()
        time.sleep_ms(220)                 # enkel sprett-demping
    sw_sist = n

tegn()
while True:
    les_dreining()
    les_trykk()
    time.sleep_ms(1)
🎛️ Slik virker dimmingen: dreieknappen gir to signaler (CLK + DT). For hvert hakk «faller» CLK; koden ser på DT for å vite retning — med klokka = lysere, mot = mørkere. Verdien lys (0–255) ganges inn i den varme grunnfargen på hele stripa. Trykk-knappen slår bare av/på og husker styrken.
